## 앱 개요
- **앱 이름**: 아쿠아캣 (AquaCat)
- **플랫폼**: iOS / Android (Flutter)
- **컨셉**: 귀여운 물고양이 캐릭터와 함께하는 물 마시기 습관 앱
- **핵심 가치**: 캐릭터 감정 변화로 동기부여 → 꾸준한 수분 섭취

## 1. 홈 화면

### 1-1. 캐릭터 영역
- 아쿠아캣 캐릭터 중앙 배치
- **수분 게이지** = 섭취량 - 시간 경과 소모량
- 현재 게이지에 따라 5단계 상태 자동 변환
  - 🥀 Critical (0~15%) — 떨림
  - 😿 Thirsty (15~40%) — 축소 + 물방울
  - 🐱 Normal (40~70%) — 숨쉬기
  - 😊 Happy (70~95%) — 바운스
  - 🎊 Perfect (95~100%) — 왕관 + 후광 + 축포
- 캐릭터 터치 시 간단 리액션 (점프/흔들)

### 1-2. 수분 게이지 시스템

**이중 표시:**
- 📊 **프로그레스 바** = 총 섭취량 / 목표량 (줄어들지 않음, 성취감)
- 🐱 **캐릭터 상태** = 게이지 기반 (시간에 따라 변동, 긴장감)

**게이지 계산:**
```
게이지(%) = (총 섭취량 - 소모량) / 목표량 × 100

소모율 = 목표량 × 30% / 활동시간
예: 목표 2,000ml, 활동 16h → 시간당 37.5ml 소모
```

**캐릭터 상태 기준:**
- 활동 시간 (기상 ~ 취침 2시간 전):
  - 🥀 Critical: 0~15%
  - 😿 Thirsty: 15~40%
  - 🐱 Normal: 40~70%
  - 😊 Happy: 70%+ (낮 최대 상태)
- 정산 시간 (취침 2시간 전 이후):
  - 🎊 Perfect: 프로그레스 100% 이상 달성 시
  - 그 외: 해당 게이지 상태 유지

**하루 사이클:**
- 기상 시간 → 게이지 0%에서 시작, 알림 시작
- 활동 중 → 물 마시면 충전 ↑, 시간 지나면 소모 ↓
- 정산 시간 (취침 -2h) → 목표 달성 시 🎊 Perfect 연출
- 취침 시간 → 알림 중단, "아쿠아캣가 잠들 준비를 해요 💤"
- 앱을 안 열어도 백그라운드 계산 (다음 열었을 때 반영)

**예시 (목표 2,000ml, 기상 08:00, 취침 24:00, 정산 22:00):**
```
08:00  섭취 0     진행률 0%    게이지 0%   🥀 Critical
09:00  섭취 300   진행률 15%   게이지 13%  🥀 Critical
10:00  섭취 600   진행률 30%   게이지 24%  😿 Thirsty
12:00  섭취 1000  진행률 50%   게이지 41%  🐱 Normal
15:00  섭취 1400  진행률 70%   게이지 49%  🐱 Normal
18:00  섭취 1800  진행률 90%   게이지 59%  🐱 Normal
20:00  섭취 2000  진행률 100%  게이지 55%  🐱 Normal
22:00  섭취 2200  진행률 110%  게이지 58%  → 정산! 🎊 Perfect!
```

### 1-3. 진행률 표시
- 원형 프로그레스 바 (캐릭터 뒤) — **현재 게이지** 기준
- 현재 섭취량 / 목표량 텍스트 (예: 1,200ml / 2,000ml)
- 게이지 퍼센트 표시 (실시간 변동)

### 1-3. 물 마시기 버튼
- 하단에 프리셋 버튼 3~4개
  - 🥛 150ml (한 컵)
  - 🥤 250ml (종이컵)
  - 🧴 350ml (텀블러)
  - 🍶 500ml (페트병)
- 커스텀 입력 버튼 (+) — 직접 ml 입력
- 버튼 탭 시 물결 애니메이션 + 수치 업데이트

## 2. 리마인더 (알림)

### 2-1. 알림 정책
- 알림 ON/OFF 토글 (전체)
- 개별 ON/OFF 가능

| # | 시점 | 조건 | 메시지 예시 |
|---|------|------|-----------|
| 1 | 🌅 기상 | 매일 기상 시간 | "{호칭}아, 좋은 아침! 물 한 잔 어때? 💧" |
| 2 | 🍽️ 아침 식후 | 기상+30분 | "{호칭}아, 아침 먹었으면 물도! 😊" |
| 3 | 🍽️ 점심 식후 | 13:00 (기본) | "{호칭}아, 점심 먹고 물 한 잔~ 🥤" |
| 4 | 🍽️ 저녁 식후 | 19:30 (기본) | "{호칭}아, 저녁 먹고 수분 보충! 💧" |
| 5 | 😿 상태 알림 | 게이지 40% 이하 | "{호칭}아, 물 마신 지 꽤 됐어~" |
| 6 | 💡 건강팁 | 기상 알림과 함께 | "{호칭}아, 아침 공복에 물 한 잔은 장 건강에 좋대! 💡" |
| 7 | 🌙 정산 | 취침 -2시간 | "{호칭}아, 오늘 하루 수고했어! 🌙" |

- 💡 건강팁은 매일 1개씩 랜덤 (기상 알림에 포함)
  - "아침 공복에 물 한 잔은 장 건강에 좋대!"
  - "물을 자주 마시면 피부가 좋아진대!"
  - "카페인 음료 마신 후엔 물을 더 마시는 게 좋아!"
  - "식사 30분 전 물 한 잔이 소화에 도움된대!"
  - "하루 8잔 물이 두통 예방에 좋대!"
  - (20~30개 사전 등록, 추후 서버에서 관리)
- 😿 상태 알림은 **최소 1시간 간격** (스팸 방지)
- 식사 시간은 설정에서 변경 가능 (기본: 08:30/13:00/19:30)
- 취침 이후: 알림 중단

## 3. 목표 설정

### 3-1. 자동 계산
- 공식: `체중(kg) × 활동계수`
  - 적음: 체중 × 30ml
  - 보통: 체중 × 33ml
  - 많음: 체중 × 40ml
- 100ml 단위 반올림
- 예시:
  - 60kg 보통 → 60×33 = 1,980 → **2,000ml**
  - 80kg 많음 → 80×40 = 3,200 → **3,200ml**
  - 112kg 보통 → 112×33 = 3,696 → **3,700ml**
- 출처: IOM/EFSA 기반 체중×30~40ml 공식

### 3-2. 목표량 표시
- 자동 계산 결과를 확인만 가능 (수동 변경 불가)
- 목표량 변경 → 체중/활동량 수정으로만 가능 (설정 > 프로필)

## 7. 데이터 저장

### 7-1. 서버 (메인)
- **Backend**: NestJS + Prisma + PostgreSQL (Railway)
- **DB 모델**:
  - `User` — id, email, provider, displayName, gender(선택), birthYear(선택), weight, wakeTime, sleepTime, activityLevel, goalMl, isPremium, createdAt
  - `WaterLog` — id, userId, amount(ml), drinkType, loggedAt
  - `DailySummary` — id, userId, date, totalMl, goalMl, gaugePercent, streak
- **API 엔드포인트**:
  - `POST /auth/google` — Google 로그인
  - `POST /auth/apple` — Apple 로그인
  - `GET /user/profile` — 프로필 조회
  - `PATCH /user/profile` — 프로필 수정 (체중, 목표 등)
  - `POST /water/log` — 물 마시기 기록
  - `DELETE /water/log/:id` — 기록 삭제
  - `GET /water/today` — 오늘 기록 + 현재 게이지
  - `GET /water/calendar?month=2026-02` — 월간 달성도
  - `GET /water/weekly` — 주간 차트 데이터
  - `GET /water/streak` — 연속 달성일

### 7-2. 오프라인
